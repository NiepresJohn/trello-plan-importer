# Trello Plan Importer

Create a **new Trello board** from an AI-generated plan. Paste a JSON plan (generated by any AI provider), review and edit it, then commit to Trello in one click. Lists and cards are created from your JSON; existing boards can be **replaced** (all existing lists archived) when you commit again to the same board.

## Overview

The Trello Plan Importer is a Next.js application that allows you to:

- Import AI-generated project plans directly into Trello boards
- Review and edit cards, labels, and lists before committing
- Replace existing board content with updated plans
- Manage multiple projects through a clean, intuitive interface

**Key Features:**

- **AI-Powered:** Works with plans generated by ChatGPT, Claude, Gemini, or any LLM
- **Draft Editor:** Review and modify cards, labels, checklists, and due dates before committing
- **One-Click Commit:** Create boards, lists, and cards in Trello with a single click
- **Replace Mode:** Update existing boards by archiving old lists and creating new ones
- **Secure:** All Trello credentials are kept server-side; never exposed to the client

## Getting Started

### Prerequisites

Before you begin, make sure you have:

- A [Trello](https://trello.com) account
- [Node.js](https://nodejs.org) (version 18 or higher)
- [Git](https://git-scm.com) installed on your machine
- A hosting platform account ([Netlify](https://netlify.com) or [Vercel](https://vercel.com))

### Get Trello API Credentials

You'll need three pieces of information to connect to Trello:

#### 1. **Trello API Key**

1. Log in to your Trello account
2. Visit the Trello Developer API Key page: **https://trello.com/power-ups/admin**
3. Click **"New"** to create a new Power-Up (or use an existing one)
4. Once created, you'll see your **API Key** displayed on the page
5. Copy and save this key securely

Alternative direct link: **https://trello.com/app-key**

#### 2. **Trello Token**

1. On the same API Key page (https://trello.com/app-key), you'll see a link that says **"Token"** or **"manually generate a Token"**
2. Click this link to authorize token generation
3. Review the permissions requested (read and write access to your boards)
4. Click **"Allow"** to generate your token
5. Copy and save the token securely

**Important:** This token gives full access to your Trello account, so treat it like a password.

#### 3. **Webhook Secret**

The webhook secret is a password you create yourself. It's used to secure API requests between your application and Trello.

- Choose a strong, random string (at least 32 characters)
- You can generate one using: `openssl rand -hex 32`
- Or use an online password generator

Example: `a7f3c9e2b4d1f8a6c3e9b2d4f1a8c6e3b9d2f4a1c8e6b3d9f2a4c1e8b6d3f9a2`

### Installation

1. **Clone the repository:**

```bash
git clone https://github.com/NiepresJohn/trello-plan-importer.git
cd trello-plan-importer
```

2. **Install dependencies:**

```bash
npm install
```

3. **Create a `.env.local` file** in the project root:

```bash
touch .env.local
```

### Environment Setup

Open `.env.local` and add your credentials:

```env
# Required: Trello API Credentials
TRELLO_KEY=your_trello_api_key_here
TRELLO_TOKEN=your_trello_token_here
WEBHOOK_SECRET=your_generated_webhook_secret_here

# Recommended: Default board and list names
DEFAULT_BOARD_NAME=My Project Board
DEFAULT_LIST_NAME=To Do

# Optional: Advanced configuration
DEFAULT_BOARD_ID=
ALLOW_BOARD_CREATE=true
ALLOW_LABEL_CREATE=true
RATE_LIMIT_PER_MINUTE=30
```

**Environment Variables Explained:**

| Variable | Required | Description |
|----------|----------|-------------|
| `TRELLO_KEY` | Yes | Your Trello API key from https://trello.com/app-key |
| `TRELLO_TOKEN` | Yes | Your Trello token (generated on the API key page) |
| `WEBHOOK_SECRET` | Yes | A secure random string you create (32+ characters) |
| `DEFAULT_BOARD_NAME` | Recommended | Default board name when not specified in JSON |
| `DEFAULT_LIST_NAME` | Recommended | Default list name for cards without a list |
| `DEFAULT_BOARD_ID` | Optional | Use a specific board ID instead of name lookup |
| `ALLOW_BOARD_CREATE` | Optional | Allow creating new boards (default: true) |
| `ALLOW_LABEL_CREATE` | Optional | Allow creating new labels (default: true) |
| `RATE_LIMIT_PER_MINUTE` | Optional | Limit API requests per minute (default: 30) |

**Run the development server:**

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) in your browser to see the app.

## Deployment

### Deploy to Netlify

Netlify is the recommended deployment platform for this project.

**Step 1: Prepare your repository**

1. Push your code to GitHub:
```bash
git remote add origin https://github.com/YourUsername/trello-plan-importer.git
git push -u origin main
```

**Step 2: Create a Netlify site**

1. Log in to [Netlify](https://app.netlify.com)
2. Click **"Add new site"** → **"Import an existing project"**
3. Choose **"GitHub"** and authorize Netlify to access your repositories
4. Select your `trello-plan-importer` repository
5. Configure build settings:
   - **Build command:** `npm run build`
   - **Publish directory:** `.next`
   - Netlify will automatically detect Next.js and apply the correct settings

**Step 3: Add environment variables**

1. In your Netlify site dashboard, go to **Site settings** → **Environment variables**
2. Click **"Add a variable"** and add each of the following:
   - `TRELLO_KEY`
   - `TRELLO_TOKEN`
   - `WEBHOOK_SECRET`
   - `DEFAULT_BOARD_NAME` (optional)
   - `DEFAULT_LIST_NAME` (optional)

3. Click **"Save"**

**Step 4: Deploy**

1. Click **"Deploy site"**
2. Wait for the build to complete (usually 1-2 minutes)
3. Your site will be live at `https://your-site-name.netlify.app`

**Step 5: (Optional) Custom domain**

1. Go to **Site settings** → **Domain management**
2. Click **"Add custom domain"**
3. Follow the instructions to configure your DNS

### Deploy to Vercel

Alternatively, you can deploy to Vercel:

1. Push your code to GitHub
2. Log in to [Vercel](https://vercel.com)
3. Click **"New Project"**
4. Import your `trello-plan-importer` repository
5. Add environment variables in the **Environment Variables** section
6. Click **"Deploy"**

## Usage Guide

### Step 1: Generate a JSON Plan

Use any AI provider (ChatGPT, Claude, Gemini, etc.) to generate a project plan in JSON format.

**Example prompt for ChatGPT/Claude:**

```
Create a Trello board plan for [YOUR PROJECT DESCRIPTION] in JSON format.

Use this structure:
{
  "board": {
    "name": "Board Name",
    "goal": "Project goal",
    "labels": {
      "priority": ["P0", "P1", "P2"],
      "type": ["Feature", "Bug", "Enhancement"]
    },
    "columns": [
      {
        "title": "To Do",
        "cards": [
          {
            "title": "Task name",
            "description": "Task details",
            "labels": ["P0", "Feature"]
          }
        ]
      }
    ]
  }
}
```

### Step 2: Import the Plan

1. Open your deployed Trello Plan Importer app
2. Paste the JSON plan into the text area
3. Click **"Load Plan"**

### Step 3: Review and Edit

The app will display your plan with all cards, lists, and labels. You can:

- Edit card titles and descriptions
- Add or remove labels
- Reorder cards between lists
- Add due dates and checklists
- Delete cards or lists

### Step 4: Commit to Trello

1. Click **"Commit to Trello"**
2. The app will:
   - Create the board if it doesn't exist
   - Archive existing lists if replacing a board
   - Create all lists (columns) from your plan
   - Create all cards with their labels, descriptions, and metadata

3. You'll receive a confirmation with a link to your Trello board

### Step 5: Replace a Board (Optional)

To update an existing board with a new plan:

1. Use the same board name in your JSON
2. Follow steps 2-4 above
3. The app will **archive all existing lists** and create new ones from your JSON

**Note:** Archived lists are not deleted; you can restore them from Trello if needed.

## JSON Structure

The plan JSON must follow this structure strictly. AI-generated plans should match this format.

### Basic Structure

```json
{
  "board": {
    "name": "Board Name (required)",
    "goal": "Optional project goal",
    "techStack": {
      "frontend": ["Next.js", "React"],
      "backend": ["Node.js", "PostgreSQL"]
    },
    "labels": {
      "priority": ["P0", "P1", "P2"],
      "type": ["Feature", "Bug", "Enhancement"]
    },
    "columns": [
      {
        "title": "Column/List Name",
        "description": "Optional column description",
        "cards": [
          {
            "title": "Card Title (required)",
            "description": "Optional card description",
            "labels": ["P0", "Feature"]
          }
        ]
      }
    ]
  }
}
```

### Field Reference

**Board Level:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Trello board name |
| `goal` | string | No | High-level project goal |
| `techStack` | object | No | Technology stack (any key-value pairs) |
| `labels` | object | No | Label categories; each value is an array of label names |
| `columns` | array | Yes | List of columns/lists |

**Column Level:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | Yes | List name (e.g., "To Do", "In Progress") |
| `description` | string | No | Column description |
| `cards` | array | Yes | List of cards in this column |

**Card Level:**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `title` | string | Yes | Card title |
| `description` | string | No | Card description/details |
| `labels` | array | No | Array of label names (must match labels defined in `board.labels`) |

### Alternative Field Names

The importer automatically normalizes these variants:

- `board.lists` → `board.columns`
- `card.name` → `card.title`
- `card.desc` → `card.description`
- `column.name` → `column.title`

## API Routes

The application exposes two server-side API routes:

### `GET /api/trello/meta`

Retrieves boards and lists from your Trello account.

**Query Parameters:**
- `boardName` (optional): Filter by board name
- `boardId` (optional): Get lists for a specific board

**Response:**
```json
{
  "boards": [
    {
      "id": "board123",
      "name": "My Board",
      "url": "https://trello.com/b/board123"
    }
  ],
  "lists": [
    {
      "id": "list456",
      "name": "To Do",
      "idBoard": "board123"
    }
  ]
}
```

### `POST /api/trello/commit`

Creates or updates a Trello board from a plan.

**Request Body:**
```json
{
  "board": {
    "name": "My Project",
    "columns": [...]
  }
}
```

**Response:**
```json
{
  "ok": true,
  "boardId": "board123",
  "boardUrl": "https://trello.com/b/board123",
  "listsCreated": 3,
  "cardsCreated": 12
}
```

## Security

### Best Practices

- **Never commit `.env` or `.env.local`** files to version control
- **Keep your Trello token secure** — it has full access to your account
- **Use a strong webhook secret** (32+ characters, random)
- **Enable HTTPS** in production (automatic on Netlify/Vercel)

### How Credentials Are Protected

- Trello API credentials (`TRELLO_KEY`, `TRELLO_TOKEN`) are only used in API routes (server-side)
- The client never receives or sends these credentials
- `WEBHOOK_SECRET` can be used to authenticate external requests (currently optional)

### Rate Limiting

The app includes optional rate limiting:

- Set `RATE_LIMIT_PER_MINUTE` to limit API requests
- Default: 30 requests per minute
- In-memory rate limiting (resets on server restart)

## Troubleshooting

### Issue: "Invalid API key or token"

**Solution:**
1. Verify your `TRELLO_KEY` and `TRELLO_TOKEN` in `.env.local`
2. Regenerate your token at https://trello.com/app-key
3. Restart your development server: `npm run dev`

### Issue: "Board not found"

**Solution:**
1. Check that the board name in your JSON matches exactly (case-sensitive)
2. Verify you have access to the board in Trello
3. Try using `DEFAULT_BOARD_ID` instead of `DEFAULT_BOARD_NAME`

### Issue: "Failed to create board"

**Solution:**
1. Check that `ALLOW_BOARD_CREATE=true` in your environment variables
2. Verify your Trello token has write permissions
3. Check Trello API rate limits (https://developer.atlassian.com/cloud/trello/guides/rest-api/rate-limits/)

### Issue: "Labels not created"

**Solution:**
1. Set `ALLOW_LABEL_CREATE=true` in your environment variables
2. Ensure label names match those defined in `board.labels`

### Issue: Build fails on Netlify

**Solution:**
1. Check build logs for specific errors
2. Verify `package.json` has all required dependencies
3. Ensure `netlify.toml` is present with correct configuration
4. Clear build cache: **Site settings** → **Build & deploy** → **Clear cache and retry deploy**

### Issue: Environment variables not working

**Solution:**
1. In Netlify: Go to **Site settings** → **Environment variables** and verify all variables are set
2. Variable changes require a new deployment to take effect
3. Click **Deploys** → **Trigger deploy** → **Deploy site**
