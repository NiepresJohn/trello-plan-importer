# Trello Plan Importer

Create a **new Trello board** from an AI-generated plan. Paste a JSON plan (generated by any AI provider), review and edit it, then commit to Trello in one click. Lists and cards are created from your JSON; existing boards can be **replaced** (all existing lists archived) when you commit again to the same board.

## Important

- **Purpose:** This tool is for **creating a new Trello board** (or replacing an existing one) from a structured plan.
- **JSON source:** The JSON file **must be generated by an AI provider** (e.g. ChatGPT, Claude, Gemini, or any LLM) and **must follow strictly** the JSON structure documented in this README. The importer normalizes common variants (e.g. `title` → `name`, `description` → `desc`, `columns` or `lists`), but the overall shape should match the sample below.

## What this app does

- Next.js (App Router) UI: paste a JSON plan, load it, then move through steps (Import → Board & List → Review Cards → Commit).
- Draft editor for cards, labels, checklists, and due dates before committing.
- API routes keep Trello credentials server-side; one-click commit creates the board (if needed), lists, and cards.
- **Replace behavior:** When you commit to an existing board, all existing lists on that board are archived first, then new lists and cards from your JSON are created.

## Architecture

- **Next.js UI:** `app/` (client-side editor)
- **API routes:** `app/api/trello/meta`, `app/api/trello/commit`

## Environment variables

Required:

- `TRELLO_KEY`
- `TRELLO_TOKEN`

- `DEFAULT_LIST_NAME`

## JSON structure (strict)

The plan JSON must have this shape so the importer can create the board and cards correctly. AI-generated plans should follow it strictly.

- **board** — Single object containing everything.
  - **board.name** — Trello board name (required). Board is created if it doesn’t exist.
  - **board.goal** — Optional. High-level goal or description.
  - **board.techStack** — Optional. Object (e.g. `frontend`, `backend`, `payments`, `hosting`, `observability`, `security`); each value is an array of strings.
  - **board.labels** — Optional. Object with keys like `priority`, `area`, `role`, `type`, `status`; each value is an array of label names. Cards reference labels by name in a **labels** array (e.g. `["P0", "Backend", "Planned"]`).
  - **board.columns** — Array of lists (required). Each column has:
    - **id** — Optional. Column identifier.
    - **title** — List name (e.g. "To Do", "In Progress", "Done").
    - **description** — Optional. Column description.
    - **cards** — Array of cards in this list. Each card has:
      - **id** — Optional. Card identifier.
      - **title** — Card title (required).
      - **description** — Optional. Card description.
      - **labels** — Optional. Array of label names (strings), e.g. `["P0", "Backend", "Planned"]`.

You can use **board.lists** instead of **board.columns**; each list can use **name** instead of **title**. Cards can use **name** instead of **title** and **desc** instead of **description**.

## Internal API routes

The UI calls these server-side routes; they talk to Trello.

- `GET /api/trello/meta` — Boards (and optional lists).
- `POST /api/trello/commit` — Create/replace board lists and cards from the plan.

## Security

- Trello credentials are only used in API routes, never in client code.
- `WEBHOOK_SECRET` is for server-to-server use only.
- Plan payloads are validated before any Trello actions.

## Deployment (Netlify)

2. Set the required environment variables in the Netlify dashboard.
